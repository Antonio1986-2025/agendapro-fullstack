# Plano: Melhoria do Agente IA para WhatsApp

## Problemas Identificados

### 1. Baixa qualidade de respostas
- **Modelo**: Usa `gpt-4o-mini` com `temperature: 0.1` e `max_tokens: 600` — respostas curtas e por vezes frias
- **Prompt**: ~2000 linhas com muitas regras conflitantes e exemplos redundantes
- **Sem classificação de intenção**: Não distingue "quero agendar" de "quero cancelar" de "dúvida geral" antes de entrar no fluxo

### 2. Perda de contexto conversacional
- **Janela de histórico**: Apenas últimas 20 mensagens (`historico.slice(-20)`) — em conversas longas, o agente perde referências anteriores
- **Prompt estático**: O system prompt não inclui resumo do histórico nem preferências do cliente
- **Sem memória de cliente**: Não consulta histórico de visitas, preferências, ou agendamentos anteriores para personalizar a conversa

### 3. Fluxo de agendamento pouco fluido
- **Slot-filling rígido**: Pergunta tudo em ordem, mesmo quando o cliente já deu várias informações de uma vez
- **Não extrai múltiplas intenções**: Se cliente diz "quero corte amanhã com o Luiz às 15h", preenche só 1 slot por iteração
- **Sem detecção de urgência**: Não prioriza respondendo dúvidas antes de cobrar dados

### 4. Sem fallback para humano
- Quando o agente não sabe responder, dá respostas genéricas como "Desculpe, problema técnico"
- Não oferece transferência para atendente humano
- Não detecta frustração do cliente

## Plano de Melhorias

### Fase 1: Melhorar o System Prompt (arquivo principal)

**Arquivo**: `server/services/ai.js` — função `montarSystemPrompt()` (linha 1714)

**Mudanças:**

1. **Adicionar classificador de intenção no topo do prompt**
   - Instruir o LLM a classificar a intenção ANTES de agir: agendar | cancelar | reagendar | dúvida | reclamação | elogio | saudação
   - Isso melhora a qualidade das respostas porque o agente sabe exatamente o que o cliente quer

2. **Adicionar seção de "Lidar com objeções e reclamações"**
   - Regras para quando o cliente reclama de preço, espera, atendimento
   - Tom empático: "Entendo sua frustração", "Vou verificar isso"
   - Quando escalar: "Vou chamar o responsável para te atender melhor"
   - Exemplos práticos de como responder a cada tipo de reclamação

3. **Adicionar instrução de extração multi-campo**
   - Quando o cliente dá várias informações de uma vez (ex: "corte amanhã 15h com Luiz"), o agente deve chamar múltiplas tools na mesma iteração
   - Exemplo: `definirServico("corte")` + `definirData("amanhã")` + `definirHorario("15h")` + `definirProfissional("Luiz")`

4. **Melhorar instruções de estilo**
   - Tornar o tom mais natural e menos robótico
   - Adicionar exemplos de como responder a saudações, agradecimentos, despedidas
   - Remover exemplos redundantes que confundem o LLM

5. **Adicionar seção de "Fallback para humano"**
   - Quando o agente não consegue resolver em 2 iterações
   - Quando cliente expressa frustração clara
   - Quando pergunta é muito complexa para o agente
   - Instrução: "Vou chamar alguém que pode te ajudar melhor com isso!"

### Fase 2: Melhorar o Motor de Tools (arquivo principal)

**Arquivo**: `server/services/ai.js`

**Mudanças:**

1. **Adicionar tool `buscarHistoricoCliente`**
   - Consulta agendamentos anteriores, preferências, e visitas do cliente
   - Permite ao agente dizer: "Vi que você já cortou aqui 3 vezes, sempre com o Luiz. Quer manter?"
   - SQL: busca `agendamentos` + `clientes.total_visitas` + último serviço contratado

2. **Adicionar tool `transferirParaHumano`**
   - Marca a conversa como "pendente_humano" no estado
   - Envia notificação ao responsável da barbearia
   - Resposta automática: "Vou chamar o responsável para te atender!"

3. **Melhorar tool `responderCliente`**
   - Adicionar parâmetro `tipo_resposta` (confirmacao | duvida | reclamacao | informacao | despedida)
   - Isso ajuda a manter o tom adequado

### Fase 3: Melhorar Contexto e Memória

**Arquivo**: `server/services/ai.js` — função `processarMensagem()` (linha 2015)

**Mudanças:**

1. **Aumentar janela de histórico**: de 20 para 30 mensagens (já é 30 no salvarConversa, inconsistente com o processarMensagem que usa 20)

2. **Enriquecer o system prompt com dados do cliente**
   - Antes de montar o prompt, buscar do banco:
     - Nome do cliente
     - Total de visitas
     - Último serviço contratado
     - Último profissional visitado
     - Agendamento próximo (se houver)
   - Incluir essas informações no prompt para personalização

3. **Aumentar max_tokens**: de 600 para 800 (respostas mais completas)

4. **Ajustar temperature**: de 0.1 para 0.3 (mais natural, menos robótico)

### Fase 4: Melhorar Parsing de Linguagem Natural

**Arquivo**: `server/services/ai.js`

**Mudanças:**

1. **Expandir `parsearData()`** (linha 71)
   - Adicionar: "próxima semana", "final de semana", "mês que vem"
   - Adicionar: "daqui a 3 dias", "daqui a 1 semana"

2. **Expandir `parsearHorario()`** (linha 155)
   - Adicionar: "meio-dia", "meia-noite", "de manhã", "de tarde"
   - Adicionar: "daqui a 2 horas", "quando puder"

3. **Expandir `resolverServico()`** (linha 203)
   - Adicionar sinônimos: "corte" → "corte masculino", "barba" → "barba"
   - Melhorar matching por similaridade (fuzzy)

### Fase 5: Webhook - Melhorar Passagem de Dados

**Arquivo**: `server/routes/whatsapp.js`

**Mudanças:**

1. **Passar `pushName` (nome do WhatsApp) para o processarMensagem**
   - Quando o cliente é novo, já usar o nome do WhatsApp para personalizar
   - Ex: "Oi [Nome]! Como posso te ajudar?" em vez de "Oi! Como posso te ajudar?"

2. **Passar imagem para o processarMensagem** (já existe mas verificar se está funcionando)
